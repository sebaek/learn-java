자바란

자바는 프로그래밍 언어이며 플랫폼이다.

자바 프로그래밍 언어

자바 프로그래밍 언어는 다음에 나열된 특징을 가진 고수준 언어이다.

  • 간단함
  • 객체지향
  • 분산
  • 멀티쓰레드
  • 동적인
  • 구조 중심
  • 이식성
  • 높은 성능
  • 강력함
  • 보안성

위 서술된 특징들은 The Java Language Environment에 설명되어 있다. James Gosling과 Henry McGilton이 작성했다.

자바 프로그래밍 언어에서 모든 소스 코드는 .java로 끝나는 텍스트 파일로 작성된다. 이 소스 파일들은 javac 컴파일러에 의해 .class 파일로 컴파일 된다. .class 파일의 내용은 우리의 프로세서로 처리 할 수 있는 코드가 들어 있지 않고 바이트코드(Java 가상 머신[Java VM]의 기계어)로 되어 있다. java 실행 도구는 우리의 프로그램을 Java 가상 머신을 통해서 실행 시킨다.

소프트웨어 개발 프로세서

Java VM은 많은 다른 운영체제 시스템에 적용 가능하기 때문에 같은 .class파일이 MS 윈도우나 솔라리스OS, 리눅스, Mac OS 등에서 실행될 수 있다. Java SE HotSpot at a Glance같은 어떤 다른 가상 머신은 퍼포먼스를 강화하기 위한 실행 중에 부가 작업을 한다. 이 작업은 병목현상을 찾거나 자주 사용되는 코드를 다시 컴파일 하는 것 등이다.

Java VM을 통해서 같은 프로그램이 여러 플랫폼에서 동작한다.

자바 플랫폼

플랫폼은 프로그램이 실행되는 하드웨어나 소프트웨어 환경이다. 위에서 MS 윈도우나 리눅스, 솔라리스OS, Mac OS 같은 유명한 플랫폼을 언급했다. 대부분의 플랫폼은 운영체제와 하드웨어의 조합이라고 얘기할 수 있다. 자바 플랫폼은 대부분의 다른 플랫폼과 다르게 소프트웨어로만 존재하는 플랫폼이다. 이 플랫폼은 하드웨어 기반의 플랫폼 위에서 실행된다.

자바 플랫폼은 두 컴포넌트로 이루어져 있다.

  • Java Virtual Machine - 자바 가상 머신
  • Java Application Programming Interface (API)

우리는 자바 가상 머신에 대해 위에서 언급했다. 이 것은 다른 하드웨어 기반 플랫폼에 이식된 자바 플랫폼에 기반이다.

API는 유용한 기능을 제공해주는 소프트웨어 컴포넌트의 큰 집합체 이다. 이 것은 관련된 클래스와 인터페이스들의 라이브러리로 묶여있다. 이 라이브러리들은 패키지라고도 부른다. 다은 섹션에서 API가 제공하는 기능들을 살펴본다.

하드웨어 위에서 API와 자바 가상 머신이 프로그램을 실행한다.

플랫폼 독립 환경에서 자바 플랫폼은 하드웨어 기본 플랫폼에 종속된 코드보다 조금 느릴 수 있다. 그러나 컴파일러와 가상 머신 기술은 이식성을 해치지 않으면ㅅ 퍼포먼스를 플랫폼 종속 코드 만큼 향상 시키고 있다.

"자바 가상 머신"과 "JVM"은 자바 플랫폼의 가상 머신을 의미한다.
