객체란?

객체는 객체지향 기술을 이해하는 데 핵심이다. 지금 주위를 둘러보면 실세계의 객체의 예시를 많이 찾을 수 있다. 당신의 개, 책상, 텔레비젼 세트, 자전거 등.

실세계에서 객체는 두 가지의 특징이 있다. 그 것들은 모두 상태와 기능을 가지고 있다는 것이다. 개는 상태(이름, 색, 종, 배고픔)와 기능(짖기, 가져오기, 꼬리 흔들기)가 있으며, 자전거도 역시 상태(현재 기어, 현재 케이던스, 현재 속도)와 기능(기어 바꾸기, 페달 케이던스 바꾸기, 브레이크 걸기)이 있다. 객체지향 프로그래밍에 실세계 객체의 상태와 기능을 적용하는 것은 객체지향 프로그래밍을 이해하는 데 좋은 출발점이 될 것이다.

잠시 시간을 갖고 당신 주위에 있는 실세계의 객체를 관찰해보자. 당신이 보는 각각의 객체에 대해 두 가지 질문을 가져보자: "이 객체의 상태는 무엇이 있을까?", "이 객체의 기능은 무엇이 있을까?". 관찰한 것을 적어보자. 당신이 한 것 처럼, 실세계의 객체들은 복잡하고 다양하다. 예를 들어 당신의 책상의 램프는 두 가지의 상태(on, off)와 두 가지 기능(켜기, 끄기)가 있을 것이다. 그러나 책상의 라디오는 더 많은 상태(on, off, 음량, 주파수)와 기능(켜기, 끄기, 볼륨 높이기, 볼륨 낮추기, 주파수 검색)이 있다. 또한 어떤 객체는 다른 객체를 포함하고 있는 것도 관찰했을 것이다. 이렇게 관찰한 실세계의 객체는 객체지향 프로그래밍 세계에 똑같이 적용된다.

프로그램에서의 객체

프로그램에서 객체의 개념은 실세계의 객체와 유사하다. 프로그램의 객체는 상태와 기능으로 이루어져 있다. 객체는 상태를 필드(어떤 프로그래밍 언어의 변수)에 저장하고 기능은 메소드(어떤 프로그래밍 언어에서는 함수)를 통해서 구현된다. 메소드는 객체의 내부 상태를 변화 시킬 수 있고 객체와 객체간 통신의 주요 메카니즘을 제공한다. 내부 상태를 감추고 객체의 메소드를 통해 상호 작용할 수 있도록 하는 것을 캡슐화라고 한다. - 객체지향 프로그래밍의 주요 원칙이다.

자전거를 예를 들어 살펴보면,

프로그램에서 객체화 시킨 자전거

자전거에는 상태(현재 속도, 현재 케이던스, 현재 기어)와 상태를 변경 시킬 수 있는 기능들이 있어서 자전거라는 객체 밖에서 이 기능들을 사용할 수 있게 한다. 예를 들어 자전거가 기어를 6단까지 가지고 있을 때, 기어 변경이라는 메소드를 사용하면 현재 기어를 1단부터 6단까지 바꿀 수 있다.

프로그램을 각각의 객체들로 구성한다면 다음과 같은 이점이 있다.

  1. 모듈화 : 각 객체의 소스 코드는 다른 객체들로부터 독립적으로 작성하고 유지보수 할 수 있다. 생성된 객체는 시스템 안에 쉽게 추가할 수 있다.
  2. 정보은닉 : 객체들간에 메소드를 통해서만 상호작용하도록 하면, 객체의 내부 구현은 숨겨진 상태로 유지할 수 있다.
  3. 코드 재사용 : 만약 객체가 이미 존재한다면 (만약 다른 개발자로부터 작성되었다면), 당신은 당신의 프로그램에 그 객체를 사용할 수 있다. 이 것은 전문가가 객체를 생성하고 테스트하고 디버깅하는 복잡한 작업을 하도록하고 당신은 그 것을 믿고 당신의 코드에 실행 시킬 수 있다.
  4. 쉬운 이식성과 디버깅 : 만약 특정 객체가 문제가 발생했다면, 당신은 그 것을 삭제하고 다른 객체로 교체할 수 있다. 이 것은 실제 세계와 유사한 메카니즘이다. 만약 볼트가 부러지면 기계 전체를 교체하지 않고 볼트만 교체하는 것처럼 말이다.
