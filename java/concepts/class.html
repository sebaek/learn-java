클래스란?

실제 세계에서 같은 종류인 각각의 객체를 쉽게 찾을 수 있다. 같은 모델인 수천 개의 자전거가 존재 한다고 생각 할 수 있다. 각각의 자전거는 같은 설계도에 의해서 같은 부품들을 사용해서 만들어 졌다. 객체지향에 따르면, 하나의 자전거는 클래스(class)의 객체(object)인 객체(instance)이다. 클래스(class)는 각각의 객체를 만들어 낼 수 있는 설계도인 것이다.

아래 Bicycle클래스는 자전거를 구현한 것 이다. 

class Bicycle {

    int cadence = 0;
    int speed = 0;
    int gear = 1;

    void changeCadence(int newValue) {
            cadence = newValue;
    }

    void changeGear(int newValue) {
            gear = newValue;
    }

    void speedUp(int increment) {
            speed = speed + increment;   
    }

    void applyBrakes(int decrement) {
            speed = speed - decrement;
    }

    void printStates() {
            System.out.println("cadence:" +
                cadence + " speed:" + 
                speed + " gear:" + gear);
    }
}

자바 언어의 문법을 처음 보았다 하더라도 이 클래스는 자전거 객체에 대해 설명했던 것을 코드로 작성한 것으로 이해하면 된다. cadence, speed, gear의 필드들은 객체의 상태를 나타낸 것이다. changeCadence, changeGear, speedUp등의 메소드는 객체 밖에서 상호 작용할 수 있는 기능이다.

아마 Bicycle클래스는 main메소드를 가지고 있지 않은 것을 보았을 것이다. 지금은 완전한 프로그램이 아니기 때문이다. 이 것은 다른 곳에서 사용할 수 있는 자전거에 설계도에 불과하다. 새로운(new) Bicycle 객체를 생성하고 사용하는 것은 당신의 프로그램의 다른 객체에게 달려있게 된다.

아래 BicycleDemo 클래스에서 두 개의 Bicycle 객체를 생성하고 그 메소드를 호출 하는 것을 시연한다. 

class BicycleDemo {
    public static void main(String[] args) {

        // Create two different 
        // Bicycle objects
        Bicycle bike1 = new Bicycle();
        Bicycle bike2 = new Bicycle();

        // Invoke methods on 
        // those objects
        bike1.changeCadence(50);
        bike1.speedUp(10);
        bike1.changeGear(2);
        bike1.printStates();

        bike2.changeCadence(50);
        bike2.speedUp(10);
        bike2.changeGear(2);
        bike2.changeCadence(40);
        bike2.speedUp(10);
        bike2.changeGear(3);
        bike2.printStates();
    }
}

이 예제는 두 개의 자전거의 페달 케이던스와 속도, 기어를 출력한다. 

cadence:50 speed:10 gear:2
cadence:40 speed:20 gear:3
