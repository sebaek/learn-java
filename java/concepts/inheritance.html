상속이란?

다른 종류의 객체들이 공통으로 가지고 있는 것이 있을 수 있다. 예를 들자면 산악 자전거, 로드 자전거, 2인용 자전거들은 모두 자전거의 특징(현재 속도, 현재 케이던스, 현재 기어)을 가지고 있다. 물론 부가적인 특징은 서로 다르게 가지고 있다. 2인용 자전거는 2개의 안장과 핸들바가 있고, 로드 바이크는 드롭 핸들바가 있으며 산악자전거는 낮은 기어와 추가된 체인 링이 있다.

객체지향 프로그래밍에서 클래스는 공통된 상태와 기능을 다른 클래스로부터 상속받을 수 있다. 이 예제에서는 Bicycle이 MountainBike, RoadBike, TandemBike의 슈퍼클래스가 된다. 자바 언어에서 어떤 클래스는 하나의 수퍼클래스를 가질 수 있다. 하나의 수퍼클래스는 여러 개의 서브클래스를 가질 수 있다.

자전거 클래스의 계층 구조(hierarchy)

서브클래스를 만드는 문법은 간단하다. 클래스 정의 시작 부분에 extends키워드와 상속할 클래스의 이름을 적어주면 된다. 

class MountainBike extends Bicycle {

    // new fields and methods defining 
    // a mountain bike would go here

}

위의 코드는 MountainBike가 Bicycle의 필드와 메소드를 갖게 되고 MountainBike만의 특징을 추가할 수 있게 한다. 이렇게 작성된 코드는 서브클래스를 읽기 쉽게 만들어준다. 그러나 슈퍼클래스에 정의된 상태와 기능을 잘 문서화 하여야 한다. 서브클래스에는 슈퍼클래스의 코드가 보이지 않기 때문이다.
