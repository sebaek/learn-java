패키지란?

패키지는 관련있는 클래스와 인터페이스를 묶어 놓은 네임스페이스이다. 컴퓨터의 폴더와 유사한 개념으로 생각할 수 있다. HTML 문서를 한 폴더에 넣고, 이미지는 다른 폴더에 그리고 스크립트나 프로그램을 또 다른 폴더에 넣는 것과 비슷하다. 자바 언어로 작성된 프로그램은 수백 수천개의 클래스로 구성될 수 있다. 그래스 관련있는 클래스와 인터페이스를 묶어서 여러 패키지에 나누어 넣게 될 것이다.

자바 플랫폼은 프로그램 작성시 사용할 수 있는 많은 클래스 라이브러리(패키지의 모음)을 제공한다. 이 라이브러리를 "Applicatoin Programming Interface" 또는 "API"라고 한다. 여기에 있는 패키지들은 일반적으로 프로그래밍할 때 필요한 작업들을 구현해 놓았다. 예를 들어, String 객체는 문자열에 관한 상태와 기능을 가지고 있으며, File객체는 프로그래머가 파일을 생성하고 지우고, 수정하고, 비교하는 등의 작업을 쉽게 할 수 있게 해준다. 또한 Socket 객체는 네트워크 소켓을 만들고 사용할 수 있게 한다. 말 그대로 수천개의 클래스가 존재하며 그 것을 사용 할 수 있다. 이 것은 프로그래머들에게 하부구조를 만드는 것보다 프로그램에 설계에 충실할 수 있도록 한다.

자바 SE 플랫폼에서 제공하는 모든 패키지, 인터페이스, 클래스, 필드, 메소드는 Java Platform API Specification에서 볼 수 있다. 보여주는 사이트이다. 페이지를 열고 즐겨찾기 해놓아라. 이 문서는 프로그래머로서 가장 중요한 문서가 될 것이다.
