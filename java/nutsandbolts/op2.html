비교, 관계, 조건 연산자

비교, 관계 연산자

비교, 관계 연산자는 어떤 피연산자가 다른 피연산자보다 값이 큰 지 , 작은 지, 같은 지, 같지 않은 지의 결과를 낸다. 이 연산자들은 익숙한 기호들일 것이다. 두 원시 자료형의 값이 같은 지 연산하는 기호는 "="이 아니고 "=="임을 기억해야 한다. 

==      equal to
!=      not equal to
>       greater than
>=      greater than or equal to
<       less than
<=      less than or equal to

아래 ComparisonDemo 프로그램은 비교연산자를 테스트 한다. 

class ComparisonDemo {

    public static void main(String[] args){
        int value1 = 1;
        int value2 = 2;
        if(value1 == value2)
            System.out.println("value1 == value2");
        if(value1 != value2)
            System.out.println("value1 != value2");
        if(value1 > value2)
            System.out.println("value1 > value2");
        if(value1 < value2)
            System.out.println("value1 < value2");
        if(value1 <= value2)
            System.out.println("value1 <= value2");
    }
}

결과는 : 

value1 != value2
value1 <  value2
value1 <= value2

조건 연산자

&&와 ||은 두 불린 값의 Conditional-AND와 Conditional-OR 연산을 한다. 이 연산자는 "short-circuiting" 연산을 한다. 즉 두번째 피연산자는 필요할 때에만 계산 하게 된다. 

&& Conditional-AND
|| Conditional-OR

아래 ConditionalDemo1 프로그램은 이 연산자들을 테스트 한다. 

class ConditionalDemo1 {

    public static void main(String[] args){
        int value1 = 1;
        int value2 = 2;
        if((value1 == 1) && (value2 == 2))
            System.out.println("value1 is 1 AND value2 is 2");
        if((value1 == 1) || (value2 == 1))
            System.out.println("value1 is 1 OR value2 is 1");
    }
}

조건 연산자 ?:는 if-then-else를 간단하게 표현한 연산자로 생각할 수 있다. 이 연산자는 피연산자가 3개이기 때문에 삼항 연산자로 불리기도 한다. 아래 예제는 "만약 someCondition이 true이면 result에 value1의 값을 할당하고, 그렇지 않으면 result에 value2의 값을 할당해라."로 읽을 수 있다.

ConditionalDemo2 프로그램은 ?:를 시연한다. 

class ConditionalDemo2 {

    public static void main(String[] args){
        int value1 = 1;
        int value2 = 2;
        int result;
        boolean someCondition = true;
        result = someCondition ? value1 : value2;

        System.out.println(result);
    }
}

만약 someCondition이 true이면 "1"을 출력한다. 만약 식이 간단하고 다른 영향이 없다면(할당 같은 연산) if-then-else보다 ?:를 사용하는 것이 읽기 쉽다.

타입(자료형) 비교 연산 instanceof

instanceof 연산자는 어떤 객체가 특정 타입인지 검사하는 연산자이다. 이 연산자는 어떤 객체가 어떤 클래스의 인스턴스인지, 또는 어떤 서브 클래스의 인스턴스인지, 또는 어떤 인터페이스를 구현한 클래스의 인스턴스인지 확인해 볼 수 있다.

InstanceofDemo 프로그램은 부모 클래스 ( Parent)와 간단한 인터페이스 ( MyInterface), 인터페이스를 구현하고 부모 클래스를 상속받은 자식 클래스 ( Child) 를 정의한다. 

class InstanceofDemo {
    public static void main(String[] args) {

        Parent obj1 = new Parent();
        Parent obj2 = new Child();

        System.out.println("obj1 instanceof Parent: "
            + (obj1 instanceof Parent));
        System.out.println("obj1 instanceof Child: "
            + (obj1 instanceof Child));
        System.out.println("obj1 instanceof MyInterface: "
            + (obj1 instanceof MyInterface));
        System.out.println("obj2 instanceof Parent: "
            + (obj2 instanceof Parent));
        System.out.println("obj2 instanceof Child: "
            + (obj2 instanceof Child));
        System.out.println("obj2 instanceof MyInterface: "
            + (obj2 instanceof MyInterface));
    }
}

class Parent {}
class Child extends Parent implements MyInterface {}
interface MyInterface {}

결과는 : 

obj1 instanceof Parent: true
obj1 instanceof Child: false
obj1 instanceof MyInterface: false
obj2 instanceof Parent: true
obj2 instanceof Child: true
obj2 instanceof MyInterface: true

instanceof연산자를 사용할 때 null은 어떤 타입의 인스턴스도 아님을 주의해야 한다.
