배열

배열은 하나의 자료형의 여러 개(고정된)의 값을 갖는 객체이다. 배열의 길이는 배열이 만들어질 때 결정된다. 만든 후에 그 길이는 고정된다. 배열의 사용을 "Hello World!" 프로그램의 main 메소드에서 본 적이 있을 것이다. 이 섹션에서 배열을 자세히 알아보자.

10개의 원소가 있는 배열

배열의 각 아이템을 원소(element)라고 부른다. 각 원소는 인덱스(index)를 사용해서 접근할 수 있다. 위 그림에서 볼 수 있듯이 인덱스는 0으로 시작한다. 9번째 원소는 8번 인덱스로 접근해야 한다.

아래 프로그램 ArrayDemo에서 정수 배열을 만들고 배열에 값을 넣고 출력하는 예를 보여준다. 

class ArrayDemo {
    public static void main(String[] args) {
        // declares an array of integers
        int[] anArray;

        // allocates memory for 10 integers
        anArray = new int[10];
            
        // initialize first element
        anArray[0] = 100;
        // initialize second element
        anArray[1] = 200;
        // and so forth
        anArray[2] = 300;
        anArray[3] = 400;
        anArray[4] = 500;
        anArray[5] = 600;
        anArray[6] = 700;
        anArray[7] = 800;
        anArray[8] = 900;
        anArray[9] = 1000;

        System.out.println("Element at index 0: "
                            + anArray[0]);
        System.out.println("Element at index 1: "
                            + anArray[1]);
        System.out.println("Element at index 2: "
                            + anArray[2]);
        System.out.println("Element at index 3: "
                            + anArray[3]);
        System.out.println("Element at index 4: "
                            + anArray[4]);
        System.out.println("Element at index 5: "
                            + anArray[5]);
        System.out.println("Element at index 6: "
                            + anArray[6]);
        System.out.println("Element at index 7: "
                            + anArray[7]);
        System.out.println("Element at index 8: "
                            + anArray[8]);
        System.out.println("Element at index 9: "
                            + anArray[9]);
    }
}

이 프로그램의 결과는 아래와 같다. 

Element at index 0: 100
Element at index 1: 200
Element at index 2: 300
Element at index 3: 400
Element at index 4: 500
Element at index 5: 600
Element at index 6: 700
Element at index 7: 800
Element at index 8: 900
Element at index 9: 1000

위의 예제는 배열의 문법을 알려주기 위해서 각 원소를 접근하는 모든 라인을 작성하였지만, 실제 프로그래밍에서는 반복문을 사용하게 될 것이다. 다양한 반복문( for, while, do-while)을 다른 섹션에서 배우게 될 것이다.

배열을 참조하는 변수 선언하기

위의 프로그램은 아래 코드와 같이 anArray라는 이름의 배열을 선언했다. 

// declares an array of integers
int[] anArray;

다른 자료형의 변수를 선언한 것처럼 배열의 선언은 배열의 타입과 배열의 이름으로 구성된다. 배열의 타입은 type[] 같이 작성하며 type은 원소의 자료형이다. [] 괄호는 변수가 배열이라고 알려주는 기호이다. 배열의 사이즈는 자료형을 구성하는 부분이 아니다. [] 괄호안이 비어 있는 이유이다. 배열의 이름은 네이밍 관습만 따른다면 어떤 것이든 될 수 있다. 다른 타입의 변수 처럼, 배열의 선언이 배열을 만드는 것이 아니다. 단순히 컴파일러에게 이 변수가 특정 자료형의 배열이라고 알려주는 것이다.

다른 타입의 배열을 선언할 수 있다. 

byte[] anArrayOfBytes;
short[] anArrayOfShorts;
long[] anArrayOfLongs;
float[] anArrayOfFloats;
double[] anArrayOfDoubles;
boolean[] anArrayOfBooleans;
char[] anArrayOfChars;
String[] anArrayOfStrings;

[] 괄호를 배열 이름 끝에 붙일 수도 있다. 

// this form is discouraged
float anArrayOfFloats[];

하지만 이 형식은 권장되지 않는다. [] 괄호는 배열 타입이라는 것을 알려주며 자료형과 나타나야하기 때문이다.

배열 생성, 초기화, 접근

배열을 생성하는 방법 중 하나는 new연산자를 사용하는 것이다. ArrayDemo의 다음 명령문은 10개의 정수 원소를 위한 메모리 공간을 확보하고 anArray 변수에 할당하는 코드이다.

만약 이 명령문이 없다면, 컴파일러는 컴파일이 실패했다는 아래와 같은 메시지를 출력할 것이다. 

ArrayDemo.java:4: Variable anArray may not have been initialized.

다음 몇 줄은 배열의 각 원소에 값을 할당하는 코드이다. 

anArray[0] = 100; // initialize first element
anArray[1] = 200; // initialize second element
anArray[2] = 300; // and so forth

배열에 각 원소는 인덱스를 통해서 접근한다. 

System.out.println("Element 1 at index 0: " + anArray[0]);
System.out.println("Element 2 at index 1: " + anArray[1]);
System.out.println("Element 3 at index 2: " + anArray[2]);

배열을 초기화 하는 단축된 문법을 사용할 수 있다. 

int[] anArray = { 
    100, 200, 300,
    400, 500, 600, 
    700, 800, 900, 1000
};

여기서 배열의 길이는 {}괄호안에서 콤마로 구분되어 제공된 값의 갯수로 결정된다.

또한 String[][] names처럼 []괄호를 여러개 사용하여 배열의 배열( 다차원 배열)을 선언할 수도 있다. 그러면 각 원소는 적절한 인덱스로 접근할 수 있다.

자바 언어에서 다차원 배열은 배열을 원소로 하는 배열이다. C나 Fortran의 배열과는 다르다. 이 결과 아래 MultiDimArrayDemo프로그램에서 보여주는 것과 같이 배열의 각 줄(row)는 다른 길이를 가질 수 있다. 

class MultiDimArrayDemo {
    public static void main(String[] args) {
        String[][] names = {
            {"Mr. ", "Mrs. ", "Ms. "},
            {"Smith", "Jones"}
        };
        // Mr. Smith
        System.out.println(names[0][0] + names[1][0]);
        // Ms. Jones
        System.out.println(names[0][2] + names[1][1]);
    }
}

실행 결과 : 

Mr. Smith
Ms. Jones

마지막으로, 모든 배열의 크기는 length 속성으로 알 수 있다. 아래 코드는 배열의 크기를 출력하는 예제이다. 

System.out.println(anArray.length);

배열 복사

배열을 다른 배열로 복사할 때 사용할 수 있는 System 클래스의 arraycopy 메소드가 있다. 

public static void arraycopy(Object src, int srcPos,
                             Object dest, int destPos, int length)

두 개의 Object 매개변수는 복사할 원본과 대상이 되는 배열이다. 세 개의 int 매개변수는 원본 배열의 시작위치와 대상이되는 배열의 시작위치, 복사할 원소의 개수를 나타낸다.

아래 ArrayCopyDemo 프로그램은 char 원소를 갖는 배열을 선언하고 단어 "decaffeinated"의 각 문자를 가지고 있다. System.arraycopy 메소드로 배열의 일부 원소를 두 번째 배열에 복사하는 예제이다. 

class ArrayCopyDemo {
    public static void main(String[] args) {
        char[] copyFrom = { 'd', 'e', 'c', 'a', 'f', 'f', 'e',
                'i', 'n', 'a', 't', 'e', 'd' };
        char[] copyTo = new char[7];

        System.arraycopy(copyFrom, 2, copyTo, 0, 7);
        System.out.println(new String(copyTo));
    }
}

결과는 : 

caffein

배열 조작

배열은 프로그래밍에서 강력하고 유용한 개념이다. 배열과 함께 많이 쓰이는 몇 가지 기능을 자바에서 제공한다. 예를 들어 위의 ArrayCopyDemo에서는 배열을 복사하는데 원본 배열의 원소를 각각 탐색하는 방법을 사용하지 않고 System의 arraycopy 메소드를 사용하는 예제를 보여주었다. 메소드를 호출하는 코드 한줄로 이런 일을 수행하게 한 것이다.

개발의 편의성을 위해 자바 SE는 배열을 조작(복사, 정렬, 검색 등)하는 몇 가지 메소드를 java.util.Arrays 클래스에서 제공해준다. 예를 들어 이전 예제를 java.util.Arrays 클래스의 copyOfRange 메소드를 사용하여 수정할 수 있다(ArrayCopyOfDemo 예제에서 보여줌). 차이점은 copyOfRange메소드를 사용할 땐 대상이 되는 배열을 메소드를 호출하기 전에 생성할 필요가 없다는 점이다. 대상이 되는 배열을 리턴하기 때문이다. 

class ArrayCopyOfDemo {
    public static void main(String[] args) {
        
        char[] copyFrom = {'d', 'e', 'c', 'a', 'f', 'f', 'e',
            'i', 'n', 'a', 't', 'e', 'd'};
            
        char[] copyTo = java.util.Arrays.copyOfRange(copyFrom, 2, 9);
        
        System.out.println(new String(copyTo));
    }
}

위 예제가 더 적은 라인의 코드를 사용했지만 결과는 같다(caffein). copyOfRange의 두번째 매개변수는 원본 배열의 시작 위치(포함)이며, 세번째 매개변수는 원본 배열의 끝 위치(제외)이다. 이 예제에서 복사되는 영역은 9번 인덱스(a가 위치한)를 포함하지 않는다.

java.util.Arrays의 다른 몇가지 유용한 기능은 아래와 같다.

  • 배열에서 특정한 값의 인덱스 얻을 수 있는 binarySearch 메소드
  • 두 개의 배열을 비교하여 같은지 알 수 있는 equals 메소드
  • 모든 인덱스의 값을 어떤 값으로 채울 수 있는 fill 메소드
  • 오름 차순으로 정렬할 수 있는 sort 메소드
