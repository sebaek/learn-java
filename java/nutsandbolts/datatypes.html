원시 데이터 타입

변수

자바 언어는 statically-typed(정적 타입)이다. 즉 모든 변수는 사용하기 전에 선언되어야 한다. 이 전에 본 것처럼 변수의 타입과 이름이 꼭 작성되어야 한다. 

int gear = 1;

이 코드는 "gear"라는 이름의 필드가 존재하고 정수형의 데이터를 저장할 수 있으며, 초기 값은 "1"이라고 프로그램에게 알려준다. 변수의 타입은 저장할 수 있는 값과 사용할 수 있는 연산자를 결정한다. int외에 자바 언어는 7개의 원시 데이터 타입을 지원한다. 원시 데이터 타입은 그 언어에의해 정해지며 이름은 예약어로 정해진다. 원시값은 다른 원시값과 상태를 공유하지 않는다. 자바에서 지원하는 8개의 원시 데이터 타입은 다음과 같다.

  • byte : byte 데이터 타입은 부호가 있는 2의 보수의 8비트 정수이다. 가장 작은 값은 -128이고 가장 큰 값은 127이다. byte 데이터 타입은 메모리가 한정된 곳에서 큰 배열를 사용할 때 메모리를 절약할 수 있다. 또한 데이터의 한계가 있다는 것을 코드에 명확히 표현해야 하는 곳에서 int 대신에 사용할 수 있다. 변수에 저장할 수 있는 값이 정해져 있다는 것을 코드에서 보여줄 수 있는 것이다.
  • short : short 데이터 타입은 부호가 있는 2의 보수의 16비트 정수이다. 최솟값은 -32,768이고 최댓값은 32,767이다. byte와 같은 가이드라인이 적용된다. 즉 메모리가 한정된 곳에서 큰 배열을 사용할 때 메모리를 절약할 수 있다.
  • int : int 데이터 타입은 부호가 있는 2의 보수의 32비트 정수이다. 최솟값은 -2 31이고 최댓값은 2 31-1이다. 자바 SE 8 이후에서는 int를 최솟값은 0이고 최댓값은 2 32-1인 부호가 없는 32비트 정수로 사용할 수 있다. int를 부호가 없는 정수로 사용하기 위해선 Integer 클래스를 사용하도록 해라. 자세한 정보는 Number 클래스 섹션을 보아라. 부호없는 정수의 산술 연산을 위해서 compareUnsigned, divideUnsigned같은 static 메소드가 Integer 클래스에 추가되었다.
  • long : long데이터 타입은 부호가 없는 2의 보수의 64비트 정수이다. 최솟값은 -2 63이고 최댓값은 2 63-1이다. 자바 SE 8 이후에서는 long 타입을 부호가 없는 64비트 정수로 사용할 수 있다. 이 때 최솟값은 0이고 최댓값은 2 64-1이다. 이 데이터 타입은 int보다 더 넓은 범위의 정수가 필요할 때 사용할 수 있다. 역시 Long클래스도 부호 없는 정수의 산술 연산을 위해 compareUnsigned, divideUnsigned 등의 메소드를 가지고 있다.
  • float : float데이터 타입은 단정밀도 32비트 IEEE 754 부동 소수점이다. 이 값의 범위를 설명하는 것은 이 강의의 영역을 넘어 가지만 자바 언어 명세서의 Floating-Point Types, Formats, and Values에 설명되어 있다. byte와 short처럼 메모리를 절약하기 위해 double 대신 사용할 수 있다. 이 데이터 타입은 통화(금액, 돈)같은 정밀한 계산에서 사용해서는 안된다. 정밀한 계산을 위해서는 java.math.BigDecimal클래스를 사용해야 한다. 수와 문자열 섹션에서 BigDecimal과 유용한 클래스에 대해 다룰 예정이다.
  • double : double 데이터 타입은 배정밀도 65비트 IEEE 754 부동 소수점이다. 표현할 수 있는 범위는 이 강의의 영역을 벗어난다. 자세한 내용은 Floating-Point Types, Formats, and Values을 참조하기 바란다. 소수를 사용할 때 기본적으로 선택되는 자료형이다. 위에 언급했듯이, 이 데이터 타입은 정밀한 계산을 요구하는 곳에서 사용해서는 안된다.
  • boolean : boolean 자료형은 두 개의 값만 가질 수 있다. true와 false이다. 이 자료형은 참/거짓 조건을 플래그 하기 위해 사용한다. 이 자료형은 하나의 비트로 표현하지만 "크기"가 정해진 것은 아니다.
  • char : char 자료형은 16비트 유니코드 문자이다. 최솟값은 ' \u0000' (또는 0) 이고 최댓값은 ' \u0000'(또는 65535)이다.

자바 언어에서는 위에 열거된 8개의 원시 자료형 외에 문자열을 위해 java.lang.String 클래스를 제공한다. 문자열을 쌍따옴표로 감싸면 자동으로 새로운 String 객체가 만들어진다. 예를 들어, String s = "this is a string"; String 객체는 수정이 불가하다. 즉 한 번 만들어진 객체의 값이 변경 될 수 없다는 것이다. 엄밀히 말하면 String타입은 원시 자료형은 아니지만 자바 언어에서 특별히 지원해 주므로 원시 자료형으로 생각할 수도 있을 것이다. 다른 섹션 에서 String에 대해 더 배우게 될 것이다.

기본값

필드를 선언할 때 항상 값을 할당할 필요는 없다. 초기값이 할당되지 않은 필드는 컴파일러에 의해 기본값을 할당 받게 된다. 이 값은 자료형에 따라 0이나 null이다. 그러나 기본값에 의존하는 것은 잘 못된 프로그램밍 습관이다.

위에 설명한 내용을 아래 표로 요약하였다.

자료형 (필드의) 기본값
byte 0
short 0
int 0
long 0L
float 0.0f
double 0.0d
char '\u0000'
String (또는 모든 객체) null
boolean false

지역변수는 조금 다르게 컴파일러는 초기화하지 않은 지역변수에 기본값을 할당하지 않는다. 만약 지역변수를 선언할 때 초기값을 할당 할 수 없다면 사용하기 전에 값을 할당하도록 해라. 초기값을 할당하지 않은 지역변수를 사용하려고 하면 컴파일 때 에러가 날 것이다.

리터럴

원시 데이터 타입의 변수를 초기화할 때 new키워드를 사용하지 않았다. 원시 데이터 타입은 언어를 구성하는 특별한 데이터 타입이다. 즉 클래스로 부터 만들어진 객체가 아니다. 리터럴은 고정된 값이며 계산을 필요로하지 않는 코드에 바로 표현되는 값 자체이다. 아래에서 보듯이 리터럴을 원시 데이터 타입 변수에 할당할 수 있다. 

boolean result = true;
char capitalC = 'C';
byte b = 100;
short s = 10000;
int i = 100000;

정수 리터럴

만약 정수 리터럴 끝에 L이나 l이 붙으면 long 타입의 정수 리터럴이 된다. 그렇지 않으면 int 타입의 정수 리터럴이다. 소문자 l보다는 대문자 L를 사용하는 것을 권장한다. 소문자 l은 숫자 1과 구분하기 힘들기 때문이다.

byte, short, int, long 같은 정수 타입의 값은 int 리터럴로 값을 줄 수 있다. long의 값은 int의 값의 범위를 넘으므로 long 리터럴로 값을 줄 수 있다. 정수 리터럴은 아래의 수 체계로 표현할 수 있다.

  • Decimal : 10진수, 0부터 9까지의 숫자로 표현하는 방식이다.
  • Hexadecimal : 16진수, 0부터 9까지와 A부터 F까지의 문자로 표현한다.
  • Binary : 2진수, 0과 1로 표현하는 방식이다. (자바 SE 7 이후부터 사용가능하다.)

보통 프로그래밍에서 10진수를 주로 사용하겠지만, 다른 진법으로 표시하기 위해서는 아래와 같이 작성해야 한다. 0x로 시작하면 16진수, 0b로 시작하면 2진수이다.

부동 소수점 리터럴

부동 소수점 리터럴은 float 타입( F나 f로 끝남)과 double 타입( D나 d로 끝남)가 있다.

부동 소수점 타입( float와 double)은 E 또는 e(유효숫자 표기법), F 또는 f(32비트 float 리터럴), D 또는 d(64비트 double 리터럴; 관습상 생략가능하며 디폴트이다.)를 사용해서 표현할 수 있다. 

double d1 = 123.4;
// same value as d1, but in scientific notation
double d2 = 1.234e2;
float f1  = 123.4f;

문자와 문자열 리터럴

char와 String 리터럴은 유니코드 문자로 표현할 수 있다. 편집기와 파일 시스템이 지원 한다면 당신의 코드에 바로 작성할 수 있다. 그렇지 않으면 ' \u0108'(Ĉ)나 " S\u00ED Se\u00F1or"(스페인어로 Sí Señor) 같이 "유니코드 이스케이프"를 사용할 수 있다. char는 항상 작은 따옴표 ''를 사용하고 String은 항상 큰 따옴표 ""를 사용한다. 유니코드 이스케이프 문자는 char나 String 리터럴 뿐만 아니라, 프로그램 어디에서도 사용할 수 있다(예를 들어 필드명).

자바 언어에서는 특별한 char나 String에서 사용 가능한 이스케이프 문자들이 있다; \b(backspace), \t(tab), \n(line feed), \f(form feed), \r(carriage return), \"(double quote), \'(single quote), \\(backslash).

그리고 모든 참조형 값으로 사용할 수 있는 null리터럴이 있다. null은 원시 자료형을 제외하고 모든 변수에 할당 할 수 있다. null로 할 수 있는 것은 별로 없다. null은 프로그램에서 어떤 객체가 사용할 수 없다는 것을 마킹할 때 주로 쓰게 된다.

마지막으로, 타입 명과 " .class" 붙인 클래스 리터럴이 있다. 예를 들어, String.class처럼 사용한다. 이 리터럴은 타입 자체를 표현하는 객체( Class 타입)를 가리킨다.

수형 리터럴에 언더스코어 문자 사용하기

자바 SE 7 이후에 , 모든 수 는 언더스코어(_)를 수형 리터럴에서 숫자 사이에 사용할 수 있게 되었다. 이 기능은 숫자를 그룹화해서 코드를 읽기 쉽게 만들어 준다.

예를 들어, 많은 숫자로 수를 표현해야 한다면 콤마나 스페이스 같은 구분점을 사용하는 효과를 내도록 3개씩 숫자를 묶어서 표현할 수 도 있는 것이다.

아래 예제가 수형 리터럴에서 언더스코어 문자를 사용하는 방법을 보여준다. 

long creditCardNumber = 1234_5678_9012_3456L;
long socialSecurityNumber = 999_99_9999L;
float pi =  3.14_15F;
long hexBytes = 0xFF_EC_DE_5E;
long hexWords = 0xCAFE_BABE;
long maxLong = 0x7fff_ffff_ffff_ffffL;
byte nybbles = 0b0010_0101;
long bytes = 0b11010010_01101001_10010100_10010010;

언더스코어는 숫자 사이에 쓸 수 있고 아래 나열된 곳에는 쓸 수 없다.

  • 수의 시작이나 끝
  • 소수점 옆
  • F나 L문자 앞에
  • 숫자의 문자가 기대되는 위치에

아래 예제 들은 유효한 것과 유효하지 않은 예제들을 보여준다. 

// Invalid: cannot put underscores
// adjacent to a decimal point
float pi1 = 3_.1415F;
// Invalid: cannot put underscores 
// adjacent to a decimal point
float pi2 = 3._1415F;
// Invalid: cannot put underscores 
// prior to an L suffix
long socialSecurityNumber1 = 999_99_9999_L;

// OK (decimal literal)
int x1 = 5_2;
// Invalid: cannot put underscores
// At the end of a literal
int x2 = 52_;
// OK (decimal literal)
int x3 = 5_______2;

// Invalid: cannot put underscores
// in the 0x radix prefix
int x4 = 0_x52;
// Invalid: cannot put underscores
// at the beginning of a number
int x5 = 0x_52;
// OK (hexadecimal literal)
int x6 = 0x5_2; 
// Invalid: cannot put underscores
// at the end of a number
int x7 = 0x52_;
