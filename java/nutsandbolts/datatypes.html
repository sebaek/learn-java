원시 데이터 타입

변수

이 전 강의에서 배웠듯이 객체는 상태를 필드(field)에 저장한다. 

int cadence = 0;
int speed = 0;
int gear = 1;

필드의 이름을 짓는 규칙이나 방법은 무엇인가? int외에 다른 데이터 타입은 무엇이 있는가? 필드는 선언할 때 초기값을 가져야 하는가? 만약 초기값을 주지 않으면 기본값이 할당 되는가? 등에 대한 질문에 대답을 이 강의에서 알아보겠다. 하지만 그 전에 알아야 할 것이 있다. 자바 언어에서 "필드"와 "변수"는 같은 뜻으로 사용된다. 자바를 처음 배우는 개발자에게 혼란을 주는 부분이다.

자바 언어에서 변수는 다음과 같은 종류가 있다.

  • 인스턴스 변수 (Non-static 필드) : 엄밀히 얘기하면 객체는 그들의 상태를 "non-static 필드"에 저장한다. 즉 static이라는 키워드가 없는 필드에 저장된다는 것이다. non-static 필드는 인스턴스 변수로 불리운다. 그 이유는 그 값이 각각의 인스턴스(객체)에 따라 다르기 때문이다. 예를 들어, 어떤 자전거의 currentSpeed는 다른 자전거의 currentSpeed의 값과 별도의 값이라는 것이다.
  • 클래스 변수 (Static 필드) : 클래스 변수는 static키워드와 함께 선언된 모든 필드를 가리킨다. 컴파일러는 클래스로 만들어지는 객체가 몇 개인지 상관없이 이 키워드가 지정된 변수에 하나의 저장공간만 만든다. 특정 종류의 자전거는 모두 같은 수의 기어를 가지고 있으므로 기어의 수를 저장하는 변수를 static으로 지정할 수 있다. static int numGears = 6;이라는 코드는 static 필드가 된다. 덧붙여서 final 키워드가 지정되면 기어의 수를 변경할 수 없게된다.
  • 지역 변수 : 객체가 상태를 필드에 저장하듯이 메소드는 자신의 상태를 임시로 지역 변수에 저장할 수 있다. 지역변수를 선언하는 문법은 필드를 선언하는 것과 유사하다(예: int count = 0;). 지역변수를 지정하는 특별한 키워드는 없고 변수를 선언하는 위치가 그 것을 결정하게 된다 - 메소드의 시작 중괄호와 종료 중괄호 사이에서 선언될 경우 지역변수가 된다. 그래서 지역변수는 그 것이 선언된 메소드 내에서만 접근이 가능하며 클래스의 다른 부분에서는 접근이 불가능하다.
  • 파라미터 : 이미 당신은 파라미터에 예제를 본 적이 있다. Bicycle클래스와 "Hello World!" 프로그램의 main메소드에서 보았다. main메소드의 시그니처를 떠올려보면 public static void main(String[] args) 이다. 여기 args변수가 이 메소드의 파라미터이다. 여기서 기억해야할 중요한 것은 파라미터는 항상 "필드"가 아니라 "변수"로 분류된다는 것이다. 이 것은 파라미터를 허용하는 다른 것(생성자나 예외 처리기)들에도 적용된다(나중에 배울 것임).

이 튜토리얼에서 필드와 변수에 대해 얘기할 때 다음의 가이드 라인을 따라서 이해하기 바란다. 만약 여기서 "일반적으로 필드"(지역 변수와 파라미터를 제외)를 얘기할 땐 간단히 "필드"라고 한다. 만약 "위에 설명한 모든 것"을 얘기할 땐 간단히 "변수"라고 할 것이다. 만약 문맥상 구분해야 한다면, 특정 용어를 사용할 것이다(static 필드나 지역변수 등). 그리고 당신은 "멤버"라는 용어를 보게 될 것이다. 타입의 필드와 메소드 중첩된 타입등을 묶어서 "멤버"라고 부른다.

이름 짓기 (Naming)

모든 프로그래밍 언어는 사용할 수 있는 이름의 종류와 작성하는 규칙 및 관습이 있으며, 자바도 다르지 않다. 변수의 이름을 짓는 규칙과 관습은 아래 요약해 두었다.

  • 변수 이름은 대소문자를 구분한다. 길이의 제한이 없는 유니코드 문자와 숫자의 조합이다. 문자나 달러 "$", 언더스코어 "_"로 시작해야 한다. 그러나 관습상 달러나 언더스코어보다 문자로 시작해야 한다. 그리고 달러는 관습상 절대 사용하지 않도록 하고 있다. 아마 당신은 자동으로 이름을 생성하는 상황에서 달러가 포함된 변수명을 볼 수도 있지만, 스스로 작성할 때는 사용하지 말아야한다. 유사한하게 언더스코어 문자도 사용이 가능하지만 권장되지 않는다. 스페이스 캐릭터는 사용할 수 없다.
  • 두 번째 문자부터는 문자, 숫자, 달러, 언더스코어 문자가 올 수 있다. 관습은 여기에도 적용된다. 이름을 지을 때 알아볼 수 없는 줄임말을 쓰지 말고 전체 단어로 짓는 것이 좋다. 그렇게 짓는 것이 코드를 좀더 읽고 이해하기 쉽게 만들어 줄 것이다. 그렇게 짓는 것이 코드 자체가 문서화 되도록한다. 예를 들어 cadence나 speed, gear가 s, c, g보다 더욱 직관적이고 알아보기 쉽다. 또한 이름을 선택할 때 키워드나 예약어를 선택하면 안된다.
  • 이름이 한 단어로만 이루어진 경우 소문자로 작성해야 한다. 만약 두 단어 이상으로 작성해야 할 경우 두 번째 이후 각 단어의 앞 글자를 대문자로 해야 한다. 예를 들어 gearRatio나 currentSpeed같이 말이다. 만약 변수에 상수(변하지 않는 값)이 저장되어야 하면 관습이 조금 다르게 적용된다. static final int NUM_GEARS = 6;과 같이 모두 대문자로 작성하고 각 단어를 언더스코어로 연결해라. 관습에 따라 언더스코어는 이 경우외에 작성하지 않는다.
