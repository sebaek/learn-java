어노테이션 타입 선언하기

많은 어노테이션이 주석을 대신할 수 있다.

어떤 회사에서 모든 클래스 몸체에 중요한 정보를 주석으로 작성한다고 가정해보자. 

public class Generation3List extends Generation2List {

    // Author: John Doe
    // Date: 3/17/2002
    // Current revision: 6
    // Last modified: 4/12/2004
    // By: Jane Doe
    // Reviewers: Alice, Bill, Cindy

    // class code goes here

}

같은 메타데이터를 어노테이션으로 추가 하기 위해서 어노테이션 타입을 먼저 작성해야 한다. 작성하는 문법은 아래와 같다. 

@interface ClassPreamble {
    String author();
    String date();
    int currentRevision() default 1;
    String lastModified() default "N/A";
    String lastModifiedBy() default "N/A";
    // Note use of array
    String[] reviewers();
}

어노테이션 타입을 정의하는 것은 인터페이스를 정의하는 것과 유사하다. interface 키워드 앞에 @를 붙인다. 어노테이션 타입은 다음 레슨에서 다루겠지만 interface 형식이다. 지금 인터페이스에 대해 이해할 필요는 없다.

위의 어노테이션 정의는 몸체에 메소드 처럼 보이는 어노테이션 타입 요소(annotation type element) 선언을 작성했다. 또한 기본값을 설정할 수도 있다.

어노테이션 타입을 정의한 후, 아래 처럼 값을 채워서 어노테이션 타입을 사용할 수 있다. 

@ClassPreamble (
    author = "John Doe",
    date = "3/17/2002",
    currentRevision = 6,
    lastModified = "4/12/2004",
    lastModifiedBy = "Jane Doe",
    // Note array notation
    reviewers = {"Alice", "Bob", "Cindy"}
)
public class Generation3List extends Generation2List {

// class code goes here

}

Note : Javadoc으로 생성하는 문서에 @ClassPreamble의 정보가 나타나게 하기 위해서, @Documented 어노테이션을 @ClassPreamble정의할 때 작성해야 한다. 

// import this to use @Documented
import java.lang.annotation.*;

@Documented
@interface ClassPreamble {

    // Annotation element definitions
    
}
