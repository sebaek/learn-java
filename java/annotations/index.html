Lesson: 어노테이션

메타데이터 형식인 어노테이션은 프로그램의 일부가 아니라 프로그램에 대한 정보이다. 어노테이션은 적용한 코드에 대한 직접적인 기능을하지 않는다.

어노테이션은 아래와 같은 사용 예가 있다.

  • 컴파일러에게 정보 제공 — 어노테이션은 컴파일러에 의해 사용되어 에러를 발견하거나 경고를 내지 않을 수 있다.

  • 컴파일 타임(compile-time)과 배포 타임(deployment-time)에서 처리 — 소프트웨어 툴이 어노테이션 정보를 활용해서 코드나 XML 파일등을 만들어 낼 수 있다.

  • 런타임(runtime)에서 처리 — 어떤 어노테이션은 실행시간에 활용될 수 있다.

이 강의는 어디서 어노테이션이 필요한지, 어떻게 적용하는지, Java SE API의 predefined 어노테이션이 무엇인지, 어떻게 타입 체크를 강화하기 위해 타입 어노테이션이 pluggable type system과 함께 사용할 수 있는지, 어떻게 중복 어노테이션을 사용하는지 설명한다.
