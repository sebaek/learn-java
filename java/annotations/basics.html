어노테이션 기본

가장 간단한 형태는 아래 코드처럼 작성한 것이다. 

@Entity

@ 문자는 다음에 오는 것이 어노테이션이라는 것을 컴파일러에게 알려준다. 아래 예제에서 어노테이션의 이름은 Override 이다. 

@Override
void mySuperMethod() { ... }

어노테이션은 요소(elements)를 가질 수 있다. 요소는 이름이 있거나 아닐 수 있으며 값들을 갖는다. 

@Author(
    name = "Benjamin Franklin",
    date = "3/27/2003"
)
class MyClass() { ... }

또는

@SuppressWarnings(value = "unchecked")
void myMethod() { ... }

만약 하나의 요소만 존재하고 이름이 value이면 이름을 생략할 수 있다. 

@SuppressWarnings("unchecked")
void myMethod() { ... }

만약 어노테이션이 요소를 갖지 않는다면 ()괄호를 생략할 수 있다. 위 @Override의 예제에서 보여주고 있다.

여러 어노테이션을 같은 선언에 사용할 수 있다. 

@Author(name = "Jane Doe")
@EBook
class MyClass { ... }

만약 어노테이션이 같은 타입이라면 이 것을 중복 어노테이션이라고 부른다. 

@Author(name = "Jane Doe")
@Author(name = "John Smith")
class MyClass { ... }

중복 어노테이션은 Java SE 8 부터 지원하기 시작했다. 자세한 내용은 Repeating Annotations를 참조하라.

어노테이션 타입은 Java SE API java.lang나 java.lang.annotation 패키지에 정의 되어 있는 타입중 하나일 수 있다. 이 전 예제에서 Override와 SuppressWarnings는 predefined Java annotations이다. 물론 직접 어노테이션 타입을 정의할 수 있다. 위 예제의 Author과 Ebook 어노테이션은 커스텀 어노테이션 타입이다.

어노테이션은 어디서 사용되는 가?

어노테이션은 선언하는 코드에 적용할 수 있다. 클래스, 필드, 메소드와 프로그램의 다른 요소를 선언할 때 사용된다. 관습에 따르면 선언문에 사용할 때 같은 줄에 작성된다.

Java SE 8 이 배포되면서 어노테이션은 타입이 사용되는 곳에 적용될 수 있게 되었다. 아래 예시가 있다.

  • Class instance creation expression: 
    new @Interned MyObject();
  • Type cast: 
    myString = (@NonNull String) str;
  • implements clause: 
    class UnmodifiableList<T> implements
    @Readonly List<@Readonly T> { ... }
  • Thrown exception declaration: 
    void monitorTemperature() throws
    @Critical TemperatureException { ... }

어노테이션의 이런 형식을 type annotation이라고 한다. 더 많은 정보는 Type Annotations and Pluggable Type Systems에서 확인할 수 있다.
