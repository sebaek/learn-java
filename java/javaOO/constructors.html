생성자

클래스는 객체를 생성할 때 호출되는 생성자를 갖는다. 생성자의 선언은 메소드의 선언과 유사하다. 생성자의 이름은 클래스의 이름과 같고 리턴타입이 없다. 예를 들어 Bicycle클래스는 하나의 생성자를 가지고 있다. 

public Bicycle(int startCadence, int startSpeed, int startGear) {
    gear = startGear;
    cadence = startCadence;
    speed = startSpeed;
}

새로운 Bicycle 객체 myBike를 만들기 위해 new 연산자를 사용해 생성자를 호출한다. 

Bicycle myBike = new Bicycle(30, 0, 8);

new Bicycle(30, 0, 8)은 객체를 위한 메모리 공간을 마련하고 그 필드를 초기화 시킨다.

Bicycle은 생성자를 하나만 갖고 있지만, 다른 생성자를 갖는 것도 가능하다. 매개변수가 없는 생성자를 가질 수도 있다. 

public Bicycle() {
    gear = 1;
    cadence = 10;
    speed = 0;
}

Bicycle yourBike = new Bicycle();는 새 Bicycle클래스 객체 yourBike를 만들기 위해 매개변수 없는 생성자를 호출한다.

두 생성자는 매개변수 리스트가 다르기 때문에 하나의 Bicycle클래스에 작성될 수 있다. 메소드 처럼 자바는 매개변수의 갯수와 자료형으로 생성자를 구분한다. 하나의 클래스에 매개변수의 갯수가 같고 자료형도 같은 생성자를 만들 수 없다.

꼭 생성자를 작성할 필요는 없다. 하지만 생성자를 작성할 때 주의해야 할 점들이 있다. 컴파일러는 자동으로 매개변수가 없는 생성자를 하나 만든다. 생성자가 없는 모든 클래스의 기본 생성자이다. 이 기본 생성자를 슈퍼클래스의 no-argument 생성자(매개변수 없는 생성자)라고 한다. 이 때 슈퍼클래스가 매개변수 없는 생성자가 없다면 컴파일러는 에러를 낸다. 그래서 프로그래머는 이 것을 확실하게 해야 한다. 만약 클래스의 슈퍼클래스가 지정되지 않으면 암묵적으로 Object클래스가 슈퍼클래스가 된다. Object클래스는 매개변수 없는 생성자를 가지고 있다.

슈퍼클래스의 생성자를 직접 사용할 수 있다. MoutainBike에서 그 것을 사용했다. 나중에 좀 더 알아보도록 하겠다.

접근지정자를 생성자를 선언할 때 사용할 수 있다. 어떤 클래스가 그 생성자를 사용할 수 있는 지 결정한다.

Note : 다른 클래스가 MyClass의 생성자를 사용할 수 없다면, MyClass클래스의 객체를 직접 만들 수 없다.
