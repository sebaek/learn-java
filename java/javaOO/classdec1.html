클래스 선언

아래의 방법으로 클래스를 선언하는 것을 보았다. 

class MyClass {
    // field, constructor, and 
    // method declarations
}

이 것이 클래스 선언이다. 클래스의 몸체({}괄호 사이)는 이 클래스로 만들어진 객체의 life cycle에 대한 모든 코드가 담겨있다. 즉 새로운 객체를 초기화하기 위한 생성자, 클래스와 객체를 위한 상태를 저장한 필드, 클래스와 객체의 기능을 구현한 메소드들이 있다.

위의 클래스 선언은 단순한 형태이다. 필수 요소로만 이루어져 있다. 클래스를 선언할 때 선언부에 슈퍼클래스의 이름, 어떤 인터페이스를 구현하는지 등의 더 많은 정보를 작성할 수 있다. 예를 들어 다음과 같다. 

class MyClass extends MySuperClass implements YourInterface {
    // field, constructor, and
    // method declarations
}

MyClass는 MySuperClass의 서브클래스이고, YourInterface 인터페이스를 구현한 것이다.

또한 public이나 private 같은 지정자를 추가할 수도 있다. public이나 private같은 지정자는 MyClass를 접근할 수 있는 클래스가 어떤 클래스인지 지정해준다. 나중에 좀 더 다루도록 하겠다. 인터페이스와 상속 강의에서 extends와 implements를 어떻게 사용하고 왜 사용하는 지 설명한다. 지금은 잠시 이 부분은 넘어가도 좋다.

일반적으로 클래스 선언은 아래 요소를 순서대로 가지고 있다.

  1. public, private같은 지정자.
  2. 클래스 이름, 관습에 의해 첫 글자는 대문자
  3. extends키워드와 부모 클래스의 이름. 하나의 클래스는 하나의 부모 클래스만 상속받을 수 있다.
  4. implements키워드와 콤마로 구분한 인터페이스 리스트. 하나의 클래스는 여러 인터페이스를 구현할 수 있다.
  5. {}괄호 안의 클래스 몸체
