클래스와 객체의 생성과 사용 요약

클래스 선언은 클래스에 이름을 붙이고 클래스 몸체를 {}괄호 안에 작성하는 것이다. 클래스 이름 앞에는 지정자가 붙을 수 있다. 클래스 몸체에는 필드, 메소드, 생성자가 있다. 클래스는 상태 정보를 저장하기 위해 필드를 사용하고, 기능을 구현하기 위해 메소드를 사용한다. 생성자는 이름이 클래스와 같고 리턴 타입이 없는 메소드와 유사하게 생겼으며 클래스의 새 객체를 초기화 한다.

클래스와 멤버는 선언할 때 public같은 접근 지정자를 사용해서 같은 방법으로 접근을 제어할 수 있다.

static키워드를 사용해서 클래스 변수와 클래스 멤버를 만들 수 있다. static으로 선언되지 않은 멤버는 인스턴스 멤버이다. 클래스 변수는 그 클래스의 모든 인스턴스가 공유하며 객체 참조와 클래스 이름으로 접근 할 수 있다. 클래스의 객체들은 각각의 인스턴스 변수를 가지며 객체 참조를 통해서면 접근할 수 있다.

new연산자와 생성자를 사용해서 클래스의 객체를 만들 수 있다. new 연산자는 만들어진 객체의 참조를 반환한다. 그 참조를 변수에 할당하거나 직접 사용할 수 있다.

인스턴스 변수와 메소드는 정의된 클래스 외부 코드에서 적합한 이름을 사용해서 참조할 수 있다. 적합한 인스턴스 변수의 이름은 다음과 같이 생겼다. 

objectReference.variableName

메소드의 적합한 이름은 다음과 같이 생겼다.

objectReference.methodName(argumentList)

또는

objectReference.methodName()

가비지 컬렉터는 사용하지 않는 객체를 자동으로 제거한다. 프로그램이 객체의 참조를 더 이상 가지고 있지 않을 때 객체는 사용되지 않는다. 참조를 null로 셋팅해서 참조를 직접 없앨 수 있다.
