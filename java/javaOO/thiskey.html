this 키워드 사용하기

인스턴스 메소드나 생성자에서, this는 현재 객체를 가리킨다 - 호출된 메소드나 생성자가 속한 객체. this 키워드를 사용해서 인스턴스 메소드나 생성자에서 현재 객체의 모든 멤버를 접근할 수 있다.

필드에서 this 사용하기

this키워드를 사용하는 가장 큰 이유는 메소드나 생성자의 매개변수가 필드를 가리기(shadow) 때문이다.

예를 들어 아래 처럼 작성된 Point 클래스를 

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}

이렇게 다시 작성할 수 있다. 

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

생성자의 각 매개변수는 객체의 필드를 가린다(shadow) - 생성자 안의 x는 생성자의 첫 번째 매개변수이다. Point필드의 x를 가리키기 위해서 this.x를 사용해야 한다.

생성자에서 this 사용하기

생성자 안에서 this키워드를 같은 클래스 내의 다른 생성자를 호출하는데 사용할 수 있다. 이렇게 사용하는 것을 explicit constructor invocation이라고 한다. 여기 또 다른 Rectangle클래스가 있다. 

public class Rectangle {
    private int x, y;
    private int width, height;
        
    public Rectangle() {
        this(0, 0, 1, 1);
    }
    public Rectangle(int width, int height) {
        this(0, 0, width, height);
    }
    public Rectangle(int x, int y, int width, int height) {
        this.x = x;
        this.y = y;
        this.width = width;
        this.height = height;
    }
    ...
}

이 클래스는 여러개의 생성자를 가지고 있다. 각 생성자는 Rectangle의 멤버변수를 초기화 한다. 생성자는 인자로 전달되지 않은 멤버 변수의 값을 기본 값으로 할당한다. 예를 들어 매개변수가 없는 생성자는 크기가 1×1이고 좌표가 0, 0인 Rectangle을 만든다. 매개변수가 2개인 생성자는 전달 받은 높이와 너비와 함께 좌표 0, 0을 4개인 생성자에 매개변수로 전달하고 호출한다. 컴파일러는 매개변수의 타입과 갯수에 기반하여 호출해야 하는 생성자를 결정한다.

다른 생성자 호출은 생성자의 첫번째 줄에 위치해야 한다.
