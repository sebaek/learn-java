메소드에서 반환하는 값

메소드는 아래 경우 중 하나라도 만족하면 메소드를 호출한 곳으로 반환한다.

  • 메소드의 명령문이 모두 완료 되었을 때
  • return키워드를 만났을 때
  • 예외(exception)이 던져졌을 때 (나중에 알아보자)

메소드 선언부에 리턴타입을 작성해야 한다. 그 값을 반환하기 위해서 메소드의 몸체 안에서, return 키워드를 사용한다.

void를 선언한 모든 메소드는 값을 반환하지 않는다. return키워드가 필요 없다. 흐름 제어 문을 종료시키거나 메소드를 종료시킬 때 간단히 다음과 같이 사용할 수 있다. 

return;

만약 void로 선언된 메소드에서 값을 리턴하려고 하면 컴파일 에러가 난다.

void로 선언하지 않은 모든 메소드는 return 키워드를 적절한 타입의 값과 같이 사용해야 한다. 아래 코드와 같이: 

return returnValue;

리턴값의 자료형은 메소드에 선언된 리턴타입과 일치해야 한다.

Rectangle클래스의 getArea()메소드는 정수를 리턴한다. 

// a method for computing the area of the rectangle
public int getArea() {
    return width * height;
}

이 메소드는 width*height의 계산 결과인 정수를 리턴한다.

getArea메소드는 원시 데이터 타입을 리턴한다. 어떤 메소드는 참조 타입을 반환할 수도 있다.

클래스나 인터페이스 반환하기

이 섹션은 조금 어려울 수 있다. 스킵하고 인터페이스와 상속에 대해 공부한 후 다시 돌아와서 읽기 바란다.

메소드가 리턴타입으로 클래스 이름을 사용하면 반환되는 객체의 타입의 클래스가 정확히 일치하거나 그의 서브클래스이어야 한다. ImaginaryNumber가 java.lang.Number의 서브 클래스라고 생각해보자. java.lang.Number는 Object의 서브 클래스이다. 아래 그림으로 표현하였다.

ImaginaryNumber의 클래스 계층

Number를 리턴하는 클래스가 있다고 가정해보면, 

public Number returnANumber() {
    ...
}

returnANumber 메소드는 ImaginaryNumber를 리턴할 수 있지만, Object를 리턴할 수는 없다. ImaginaryNumber는 Number의 서븤르래스 이기 때문에 ImaginaryNumber는 Number이다. 하지만 Object는 꼭 Number라고 할 수 없다 - String이나 다른 타입이 될 수 있다.

원래 메소드의 리턴타입을 그 서브클래스로 다시 정의해서 오버라이드(override)할 수 있다. 아래와 같이 : 

public ImaginaryNumber returnANumber() {
    ...
}
    이 것을 covariant return type이라고 한다. 
    즉, 리턴 타입은 같은 방향의 서브클래스로서 다양하게 존재 할 수 있다.

Note : 또한 리턴타입으로 인터페이스를 사용할 수 있다. 이 경우 리턴되는 객체는 그 인터페이스를 구현한 타입의 객체이어야 한다.
