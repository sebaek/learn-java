클래스의 멤버 접근 제어하기

접근 지정자는 특정 필드를 사용하거나 특정 메소드를 사용할 수 있는 다른 클래스를 결정한다. 두 가지 레벨의 접근 제어가 있다.

  • At the top level - public 또는 package-private(지정자 없음)
  • At the member level - public, private, protected 또는 package-private(지정자 없음)

클래스는 다른 모든 클래스가 이 클래스를 사용할 수 있도록 public으로 선언될 것이다. 만약 접근지정자 없이 클래스가 선언되면 (기본, 또는 package-private), 그 클래스는 자신의 패키지 안에서만 접근 가능하다. 패키지는 관련있는 클래스들을 묶은 그룹이며 나중에 더 배우게 될 것이다.

member level에서, top-level 클래스처럼 public이나 package-private을 사용할 수 있으며 의미도 같다. 멤버에는 두 개의 접근지정자 private와 protected가 더 있다. private 지정자는 멤버가 자신의 클래스 안에서만 접근이 가능하다고 지정한다. protected 지정자는 멤버가 자신의 패키지 안에서와 다른 패키지의 서브클래스에서 접근가능하다고 지정한다.

아래는 각 지정자가 멤버에 접근을 지정하는 표이다.

Access Levels
Modifier Class Package Subclass World
public Y Y Y Y
protected Y Y Y N
no modifier Y Y N N
private Y N N N

첫번째 데이터 열은 클래스 자신이 access level로 정의된 멤버를 접근 가능한지 여부를 보여준다. 표에서 보듯이 클래스 자신은 항상 자기 자신의 멤버에 접근 가능하다. 두번째 열은 같은 패키지의 클래스가 그 멤버에 접근 가능한지 여부를 나타낸다. 세번째 열은 이 패키지 밖의 서브클래스가 그 멤버에 접근 가능한지 여부를 보여준다. 네번째 열은 모든 클래스가 그 멤버에 접근 가능하지 알려준다.

access level은 두 가지 방법으로 영향을 미친다. 첫번째 프로그래머가 외부 소스의 클래스를 사용할 때 access level은 프로그래머의 클래스가 사용할 수 있는 외부 클래스의 멤버가 무엇인지 알려준다. 두번째는 프로그래머가 클래스를 작성할 때 , 모든 멤버 변수와 모든 메소드가 어떤 access level을 가져야 하는 지 결정해야 한다는 것이다.

여러 클래스를 가지고 access level이 visibility에 어떻게 영향을 미치는 지 보자. 아래 그림은 네 개의 클래스가 어떻게 관련 있는지 보여준다.

Classes and Packages of the Example Used to Illustrate Access Levels

Access Level을 선택하는 팁 :
만약 다른 프로그래머가 당신의 클래스를 사용할 때, 잘못된 사용으로 에러를 발생 시키고 싶지 않다면 아래 내용이 도움이 될 것이다.

  • 각 멤버에 가장 합당하고 제한된 access level을 사용해라. 특별히 아닐 이유가 없다면 private을 사용해라.
  • 상수를 제외하고 public 필드를 사용하지 않는다. (튜토리얼의 많은 예제에서 public 필드를 사용하고 있지만 간결한 예제를 보여주기 위한 것일 뿐이다. 실제 코드에선 사용을 권하지 않는다.) public 필드는 특정 구현에 얽매이게 하고 코드를 변경하는 데 유연성을 저하시킨다.
