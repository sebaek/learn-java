중첩 클래스, 지역 클래스, 무명 클래스, 람다 표현식을 사용할 때

Nested Classes에서 설명한 것 처럼 중첩 클래스는 한 번만 사용하는 클래스들을 한 곳에 두고 논리적 그룹으로 묶을 수 있게 한다. 그래서 캡슐화되며 가독성이 증가하고 유지보수성이 높아진다. 지역 클래스, 무명 클래스, 람다 표현식 역시 이런 이점을 제공한다. 그러나 이 것들은 좀더 특별한 상황에서 사용하도록 되어 있다.

  • 지역 클래스(Local class) : 어떤 클래스의 객체가 하나 이상 필요하거나 그 생성자에 접근하거나 새로운 이름 있는 타입을 만들 때 (예를 들어, 나중에 추가 메소드들을 호출할 필요가 있기 때문에) 사용하라.

  • 무명 클래스(Anonymous class) : 필드를 선언하거나 부가적인 메소드들이 필요할 때 사용하라.

  • 람다 표현식(Lambda expression) :

    • 다른 코드로 기능이 하나만 있는 단위를 전달할 때 캡슐화하기 위해 사용하라. 예를 들어, 콜렉션의 각 요소에 어떤 액션을 하거나, 프로세스가 완료되거나, 프로세스가 에러를 냈을 때 람다 표현식을 사용할 것이다.

    • 앞서 다른 조건이 필요 없고 functional interface 의 간단한 객체가 필요할 때(예를 들어, 생성자나 타입의 이름, 필드, 다른 메소드가 필요 없을 때) 사용하라.

  • 중첩 클래스(Nested class) : 지역 클래스와 유사한 요구가 있을 때 사용하라. 좀더 넓은 범위에서 사용하길 원할 때, 지역 변수나 메소드 파라미터에서만 사용 하기를 원하지 않을 때 사용하라.

    • 만약 public이 아닌 인스턴스 필드와 메소드에서 접근이 필요하면 non-static 중첩 클래스를 사용하라. static 중첩 클래스는 이런 접근이 필요없을 때 사용하라.
