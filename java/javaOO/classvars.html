클래스 멤버의 이해

이 섹션에서 필드와 메소드를 인스턴스 소속이 아닌 클래스 소속으로 만드는 static키워드에 대해 알아본다.

클래스 변수

같은 클래스 설계도에 의해 여러 객체가 만들어 질 때, 각 객체는 다른 객체와 별개로 인스턴스 변수를 가진다. Bicycle클래스에서 cadence, gear, speed는 각 Bicycle객체가 갖게 되는 자신만의 변수이다. 서로 다른 메모리 공간에 저장된다.

어떤 때에는 모든 객체를 위한 공통 변수를 원할 때가 있다. 이 것은 static 지정자로 만들 수 있다. static 지정자로 필드를 선언 하면 static 필드 또는 클래스 변수라고 부른다. 이 필드는 어떤 객체가 아니라 클래스와 관련된 필드가 된다. 클래스의 모든 인스턴스는 클래스 변수를 공유하고 메모리도 하나의 공간만 지정된다. 모든 객체는 클래스 변수의 값을 바꿀 수 있으며, 심지어 클래스의 객체를 만들지 않고도 값을 변경할 수 있다.

예를 들어, Bicycle클래스의 객체를 여러 개 만들 때 각 객체에 1부터 시작하는 시리얼 넘버를 할당해야 한다고 가정하자. 이 ID 넘버는 각 객체에 유일한 값이기 때문에 인스턴스 변수가 된다. 이 때 Bicycle 객체가 몇 개 만들어 졌는 지 알고 싶고 다음 만들어 지는 객체에 ID 넘버를 다음 번호로 할당해야 한다고 생각해보자. 이 필드는 각 객체가 아니라 클래스 전체에 관련된 값이다. 이 때 아래 작성한 코드와 같이 numberOfBicycles라는 클래스 변수가 필요하다. 

public class Bicycle {

    private int cadence;
    private int gear;
    private int speed;
        
    // add an instance variable for the object ID
    private int id;
    
    // add a class variable for the
    // number of Bicycle objects instantiated
    private static int numberOfBicycles = 0;
        ...
}

클래스 변수는 클래스 이름으로 참조 할 수 있다. 

Bicycle.numberOfBicycles

이 코드가 클래스 변수라는 것을 확실히 보여준다.

Note : static 필드를 참조 변수를 사용해서 접근 할 수도 있다.
myBike.numberOfBicycles
하지만 이 방식은 클래스 변수라는 것을 명확히 보여주지 못하므로 권장하지 않는다.

Bicycle의 생성자를 사용해서 인스턴스 변수 id의 값을 numberOfBicycles에 1 더한 값으로 할당할 수 있다. 

public class Bicycle {

    private int cadence;
    private int gear;
    private int speed;
    private int id;
    private static int numberOfBicycles = 0;
        
    public Bicycle(int startCadence, int startSpeed, int startGear){
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        // increment number of Bicycles
        // and assign ID number
        id = ++numberOfBicycles;
    }

    // new method to return the ID instance variable
    public int getID() {
        return id;
    }
        ...
}

클래스 메소드

자바에서는 static 변수처럼 static 메소드도 제공한다. static 메소드는 static 지정자로 선언되며, 객체를 만들 필요없이 클래스 이름으로 호출되는 메소드이다. 

ClassName.methodName(args)

Note : static 메소드도 객체의 참조를 이용해서 호출 할 수 있다.
instanceName.methodName(args)
하지만 클래스 메소드라는 것을 명확히 보여주지 못하므로 권장되지 안않는다.

보통 static 메소드를 사용하는 이유는 static 필드에 접근하기 위해서이다. 예를 들어 static 필드 numberOfBicycles에 접근하기 위한 메소드를 Bicycle 클래스에 추가할 수 있다.

public static int getNumberOfBicycles() {
    return numberOfBicycles;
}

인스턴스와 클래스, 변수와 메소드의 모든 조합이 가능한 것은 아니다.

  • 인스턴스 메소드는 인스턴스 변수와 인스턴스 메소드를 직접 접근할 수 있다.
  • 인스턴스 메소드는 클래스 변수와 클래스 메소드를 직접 접근할 수 있다.
  • 클래스 메소드는 클래스 변수와 클래스 메소드를 직접 접근할 수 있다.
  • 클래스 메소드는 인스턴스 변수와 인스턴스 메소드를 직접 접근 할 수 없다 - 객체의 참조를 통해 접근해야 한다. 역시 클래스 메소드는 this가 가리키는 객체가 없으므로 this키워드를 사용할 수 없다.

상수

static 지정자와 final 지정자를 함께 써서 상수를 지정하는데 사용한다. final 지정자는 필드의 값이 바뀌지 않는다는 것을 의미한다.

예를 들어, 아래 변수는 원주율 상수 PI를 선언한 것이다. 

static final double PI = 3.141592653589793;

이렇게 정의된 상수는 값을 다시 할당하지 못한다. 만약 그런 코드가 있다면 컴파일 타임에 에러가 난다. 관습에 의해 상수의 이름은 대문자로 작성한다. 이름이 한 단어 이상으로 이루어져 있으면 각 단어를 언더스코어(_)로 구분한다.

Note : 만약 원시 자료형이나 문자열이 상수로 지정되고 컴파일 타임에 값을 알 수 있으면, 컴파일러는 상수가 쓰이는 모든 코드를 그 값으로 바꾼다. 이 것을 compile-time constant라고 한다. 만약 외부에 있는 상수의 값을 바꾼다면 그 상수의 값을 사용하고 있는 다른 모든 클래스를 다시 컴파일해야 한다.

Bicycle 클래스

이 섹션을 마치고 난 후, Bicycle클래스는 아래와 같다. 

public class Bicycle {

    private int cadence;
    private int gear;
    private int speed;
        
    private int id;
    
    private static int numberOfBicycles = 0;

        
    public Bicycle(int startCadence,
                    int startSpeed,
                    int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        id = ++numberOfBicycles;
    }

    public int getID() {
        return id;
    }

    public static int getNumberOfBicycles() {
        return numberOfBicycles;
    }

    public int getCadence() {
        return cadence;
    }
        
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public int getGear(){
        return gear;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public int getSpeed() {
        return speed;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
}
