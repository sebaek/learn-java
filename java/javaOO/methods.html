메소드 정의

아래 전형적인 메소드 선언 방법을 보여준다. 

public double calculateAnswer(double wingSpan, int numberOfEngines,
                              double length, double grossTons) {
    //do the calculation here
}

메소드 선언에서의 필수 요소는 메소드의 리턴타입, 이름, ()괄호, {} 괄호와 그 안의 몸체이다.

일반적으로 메소드 선언은 6가지 요소가 순서대로 이루어져 있다.

  1. 지정자 - public, private과 나중에 배울 지정자들.
  2. 리턴타입 - 메소드가 반환(return)하는 값의 자료형 또는 반환값이 없다면 void
  3. 메소드명 - 필드명 작성에 적용되는 규칙이 메소드에도 적용되지만 관습상 조금 다른 것이 있다.
  4. () 괄호안의 매개변수 리스트 - () 괄호안에 콤마로 구분한 매개변수(자료형을 앞에 작성)의 리스트. 매개변수가 없다면 빈 () 괄호를 작성해야 한다.
  5. 예외 리스트 - 나중에 강의한다.
  6. {}로 감싼 메소드 몸체 - 지역변수의 선언등을 포함하는 메소드의 코드

지정자, 리턴타입, 매개변수는 이 강의에서 설명하고, 예외(exception)은 다른 강의에서 알아보도록 하겠다.

Definition : 메소드 선언부의 두가지 요소(메소드명과 파라미터의 자료형)를 메소드 시그니쳐(method signature)라고 한다.

위의 메소드 시그니쳐는 : 

calculateAnswer(double, int, double, double)

메소드 이름 짓기

적합하 규칙에 따라 메소드 이름을 작성하여도 관습에 따라 메소드 이름에 제한이 있을 수 있다. 관습에 따르면 메소드명은 소문자와 동사로 시작하여야 한다. 뒤에 형용사나 명사가 따라올 수 있다. 여러 단어가 조합되면 두 번째 이후의 단어들은 각 단어의 첫 글자를 대문자로 한다. 

run
runFast
getBackground
getFinalData
compareTo
setX
isEmpty

보통 클래스 안에서 메소드는 유일한 이름을 갖는다. 하지만 메소드 오버로딩(method overloading)에 의해 어떤 메소드는 다른 메소드와 이름이 같을 수 있다.

Overloading Methods

자바 언어는 메소드 overloading을 제공한다. 그리고 자바는 메소드 시그니처가 다른 메소드를 구분한다. 즉 하나의 클래스가 다른 매개변수 리스트를 갖는 같은 이름의 메소드를 가질 수 있다는 뜻이다. 다른 강의에서 좀 더 다루도록 하겠다.

다양한 타입의 데이터(문자열, 정수 등)를 캘리그래피 할 수 있는 클래스가 있다고 생각해보자. 그리고 이 클래스는 각 자료형에 대한 메소드를 가지고 있다고 가정하자. 각 메소드가 drawString, drawInteger, drawFloat 같이 다른 이름을 갖는다면 클래스를 다루기가 힘들 것이다. 자바에선 다른 매개변수 리스트를 갖는다면 그림을 그리는 메소드에 같은 이름을 사용할 수 있다. 그리서 이 클래스는 다른 매개변수를 갖는 draw라는 이름의 메소드가 4개 존재하게될 거이다. 

public class DataArtist {
    ...
    public void draw(String s) {
        ...
    }
    public void draw(int i) {
        ...
    }
    public void draw(double f) {
        ...
    }
    public void draw(int i, double f) {
        ...
    }
}

오버로드된 메소드들은 매개변수의 자료형과 갯수의 차이가 있다. 위의 예제에서 draw(String s)와 draw(int i)는 매개변수 타입이 다르기 때문에 서로 다른 메소드 이다.

매개변수의 갯수와 타입이 동일한 같은 이름의 메소드를 여러개 작성할 수 없다.

컴파일러는 리턴타입의 차이로 메소드를 구분하지 않는다. 그래서 메소드 시그니쳐가 같지만 리턴타입이 다른 메소드를 같이 작성할 수 없다.

Note : 메소드 오버로딩은 코드의 가독성을 떨어뜨릴 수 있으므로 신중하게 사용하여야 한다.
