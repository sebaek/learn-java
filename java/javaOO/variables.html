멤버 변수 선언

여러 종류의 변수가 있다.

  • 클래스의 멤버 변수 - 필드라고 부른다.
  • 코드 블럭이나 메소드의 변수 - 지역변수라고 한다.
  • 메소드 선언부의 변수 - 매개변수라고 한다.

Bicycle클래스에서 필드를 선언하기 위해 아래 코드를 사용했다. 

public int cadence;
public int gear;
public int speed;

필드는 순서대로 세가지 요소로 이루어져있다.

  1. public이나 private같은 0개 이상의 지정자
  2. 필드의 자료형
  3. 필드의 이름

Bicycle클래스의 필드는 cadence, gear, speed이고 모두 int타입이다. public키워드는 다른 모든 객체가 이 멤버에 접근 가능한 퍼블릭 멤버라고 지정한다.

접근 지정자

가장 왼쪽에 작성된 지정자는 멤버 필드에 접근 가능한 클래스가 무엇인지 지정해준다. 지금은 public과 private만 얘기해보자. 다른 접근지정자는 나중에 강의하겠다.

  • public 지정자 - 모든 클래스가 접근 가능한 필드이다.
  • private - 자기 자신만 접근 가능한 필드이다.

객체지향 프로그래밍의 캡슐화 개념에 의해 필드를 private으로 지정해주어야 한다. 즉 필드는 Bicycle클래스만 직접 접근이 가능하다는 것을 의미한다. 이 필드에 대한 접근이 필요하기 때문에, public 메소드를 활용해서 간접적으로 접근할 수 있도록 한다. 

public class Bicycle {

    private int cadence;
    private int gear;
    private int speed;
        
    public Bicycle(int startCadence, int startSpeed, int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;
    }
        
    public int getCadence() {
        return cadence;
    }
        
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public int getGear() {
        return gear;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public int getSpeed() {
        return speed;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
}

자료형

모든 변수는 자료형을 갖는다. int, float, boolean등의 원시 자료형이나 String이나 배열, 객체 같은 참조 자료형을 사용할 수 있다.

변수 명

필드, 지역변수, 매개변수등 모든 변수는 아래 네이밍 규칙과 관습을 따른다.

이 강의에서는 메소드와 클래스 이름도 같은 규칙과 관습을 갖는다. 하지만 아래 예외가 있다.

  • 클래스의 이름의 첫 글자는 대문자이어야 한다.
  • 메소드의 첫 단어는 동사(verb)이어야 한다.
